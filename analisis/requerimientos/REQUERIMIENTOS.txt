PLATAFORMA
	DEBE SER UNA WEB
	DEBE TENER LOGIN
	DEBE TENER BASE DE DATOS



ADMINISTRACION
	SE DEBE PODER CREAR USUARIOS
	SE DEBE PODER DESABILITAR USUARIOS
	SE DEBE PODER MODIFICAR PERMISOS DE USUARIO
	DEBEN EXISTIR PERMISOS DE VISUALIZACION POR MODULO
	DEBEN EXISTIR PERMISOS DE ADMINISTRACION POR MODULO
	CADA USUARIO DEBE PODER CAMBIAR SU PROPIA CONTRASEÑA
	CADA USUARIO DEBE ESTAR ASOCIADO A UN CORREO ELECTRONICO
	SE DEBE PODER SOLICITAR UNA CONTRASEÑA AUTOMATICA AL CORREO DEL USUARIO PARA MODIFICAR SU CONTRASEÑA



INVENTARIO
	DEBE MOSTRAR UNA LISTA DE PRODUCTOS
	LA LISTA DE PRODUCTOS DEBE CONTENER STOCK Y LA ULTIMA FECHA DE ACTUALIZACION
	SE DEBE PODER AGREGAR PRODUCTOS
	SE DEBE PODER ELIMINAR PRODUCTOS
	SE DEBE PODER MODIFICAR EL STOCK DE PRODUCTOS



URI
	SE DEBEN PODER CREAR URI
	SE DEBEN PODER ELIMINAR URI
	LAS URI DEBEN LIGARSE A UN PRODUCTO
	LAS URI DEBEN TENER UNA FUNCIONALIDAD LIGADA
	FUNCIONALIDAD 1 - SUMAR STOCK
	FUNCIONALIDAD 2 - RESTAR STOCK
	LAS URI DEBEN RECIBIR UN PARAMETRO DE TIPO INT EN SU ESTRUTURA
	LAS URI NO DEBEN PODER MODIFICARSE UNA VEZ CREADAS



ALTERTA
	SE DEBEN PODER CREAR ALERTAS
	LAS ALERTAS SE DEBEN PODER ELIMINAR
	LAS ALERTAS DEBEN LIGARSE AL STOCK DE UN PRODUCTO
	LAS ALERTAS DEBEN LIGARSE UNA CONDICION
	CONDICION 1 - MAYOR QUE
	CONDICION 2 - MENOR QUE
	LAS ALERTAS DEBEN ENVIARSE A NOTIFICACIONES CUANDO LA CONDICION SEA VERDADERA
	LAS ALERTAS NO DEBEN PODER SER MODIFICADAS UNA VEZ CREADAS



NOTIFICACIONES
	DEBE EXISTIR UNA SECCION DE NOTIFICACIONES
	DEBE EXISTIR UNA SUB-SECCION DE NOTIFICACIONES LEIDAS
	LAS NOTIFICACIONES DEBEN PODER MARCARSE COMO LEIDAS
	LAS NOTIFICACIONES DEBEN SER ALERTAS
	LAS NOTIFICACIONES NO DEBEN PODER SER BORRADAS



REGISTROS
	EL REGISTRO DEBE CONTENER NOMBRE DE USUARIO REPONSABLE
	EL REGISTRO DEBE CONTENER TIPO DE INTERACCION
	EL REGISTRO DEBE CONTENER FECHA
	DEBE EXISTIR UN REGISTRO POR MODULO CON TODAS LAS INTERACCIONES DE SISTEMA
	LOS REGISTROS NO DEBEN PODER SER MODIFICADOS NI ELIMINADOS



REPORTES
	LOS REPORTES DEBEN SER MENSUALES O MANUALES
	LOS REPORTES DEBEN CONTENER LOS REGISTROS DE INTERACCION ORDENADOS POR FECHA
	LOS REPORTES MENSUALES DEBEN GENERARSE AUTOMATICAMENTE AL TERMINAR EL MES
	LOS REPORTES MENSUALES DEBEN GENERARSE DE FORMA AUTOMATICA
	LOS REPORTES DEBEN ALMACENARSE
	LOS REPORTES NO DEBEN PODER MODIFICARSE
	LOS REPORTES DEBEN PODER DESCARGARSE
	DEBE EXISTIR UNA INTERFAZ QUE PERMITA GENERAR REPORTES DE FORMA MANUAL
	SE DEBE PODER SELECCIONAR FECHA DE INICIO Y FIN DE REPORTES MANUALES
	LA FECHA DEBE CONTENER DIA MES AÑO HORA MINUTO SEGUNDO
	LOS REPORTES DEBEN CONTENER TODAS LAS INTERACCIONES DE LOS REGISTROS
	LOS REPORTES DEBEN MOSTRAR LA CANTIDAD DE PRODUCTOS INICIAL Y FINAL Y SU STOCK AL INICIO Y FIN DEL PERIODO DE REPORTE
	LOS REPORTES DEBEN MOSTRAR EL MOVIMIENTO DE STOCK TOTAL POR PRODUCTO DURANTE EL PERIODO DESIGNADO
	LOS REPORTES DEBEN MOSTRAR EL NOMBRE FECHA Y REPOSPONSABLES DE CREACION DE PRODUCTOS DURANTE SU PERIODO
	LOS REPORTES DEBEN MOSTRAR EL NOMBRE FECHA Y REPOSPONSABLES DE ELIMINACION DE PRODUCTOS DURANTE SU PERIODO
	LOS REPORTES NO DEBEN PODER ELIMINARSE
